ARBITRARILY_HIERARCHICAL_STRUCTURE

Axioms

Let PERSON be a data object template which represents exactly one unique human instance (whether than humanoid is alive or else dead).

Let MIND be a data object template which represents exactly one unique universe instance (which contains all instances of PERSON simultaneously).

Let Z be some finite instance of MIND (which is constrained to at least two subjective perspectives: X and Y).

Let X and Y each be PERSON objects which interact with each other inside of the same contextual instance of MIND, Z.

Set X to have more importance with respect to Z than Y does by implementing the instruction, Z.promote_former_demote_latter(X,Y).

Implications

It can be assumed that X will get more time to broadcast its thoughts through Z than will Y.

In other words, X.total_thought_transmission is a higher number than is Y.total_thought_transmission.

Not only does X get to use up more bandwidth inside of Z at any given point in time than is Y, X is also more likely to have a stable and robust identity established inside of Z’s long-term memory than is Y due to finite resource allocation which is deterministically constrained by the prioritization rules decided by Z for the autonomous dynamic closed-system of perpetual information flow which is Z.
